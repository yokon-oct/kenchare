# 要件定義書：都道府県当てWebアプリ

**作成日：** 2026年3月23日  
**バージョン：** 1.0

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | 未定（例：「けんちゃれ！」など） |
| 目的 | 都道府県の特徴をヒントに、日本地図上の場所を当てる学習型ゲームアプリ |
| 対象ユーザー | 一般向け（10代〜大人） |
| 言語 | 日本語のみ |
| デザインテイスト | ポップでシンプル |

## 3. 機能要件

### 3-1. 認証機能

- メールアドレス＋パスワードによる新規登録・ログイン
- ログアウト
- 未ログインでもゲームのプレイは可能
- スコアの保存・ランキングへの掲載はログインユーザーのみ

### 3-2. ゲーム機能

#### ゲームフロー

```
難易度選択
  ↓
ゲーム開始（10問）
  ↓
ヒント表示（1問につき1ヒント・30秒タイマー開始）
  ↓
地図上の都道府県をクリック/タップして回答
  ↓
正誤判定 → スコア加算
  ↓
次の問題へ（×10問繰り返し）
  ↓
結果画面（合計スコア・正解数・ベストスコア更新判定）
```

#### 出題仕様

- 1ゲームにつき10問固定
- 47都道府県からランダムで出題（同一ゲーム内で重複なし）
- 1問につきヒントを1枚表示
- タイマーは1問ごとに30秒
- 地図上の都道府県をクリック/タップして回答
- 早く正解するほどボーナス点が加算される

#### 難易度区分

| 難易度 | ヒント内容の例 | 基本点 |
|--------|--------------|--------|
| かんたん | 県庁所在地、有名な食べ物・名産品、代表的な観光地 | 100点 |
| ふつう | 地理的特徴、伝統工芸、方言・地域文化 | 200点 |
| むずかしい | 人口・面積ランキング、マニアックな歴史・雑学 | 300点 |

#### スコア計算方式

- **獲得点 = 基本点 × 時間ボーナス係数**
  - 残り時間が多いほど係数が高くなる（最大2.0倍、タイムアップ直前は最小1.0倍）
- 不正解・タイムアップは0点
- 1ゲームの合計スコア = 10問分の獲得点の合計

**理論上の最大スコア例**

| 難易度 | 最大スコア（10問・全問即答正解） |
|--------|-------------------------------|
| かんたん | 2,000点 |
| ふつう | 4,000点 |
| むずかしい | 6,000点 |

### 3-3. 地図機能

- SVGベースの日本地図を画面に表示
- 各都道府県の領域がクリック/タップ可能
- ホバー時にハイライト表示（PC）
- 回答後に以下のフィードバックを視覚的に表示
  - 正解：正解した都道府県を緑色でハイライト
  - 不正解：クリックした箇所を赤色、正解の都道府県を緑色でハイライト

### 3-4. スコア・ランキング機能

- **マイベストスコア**：ログインユーザー自身の難易度別ベストスコアを保存・表示
- **全体ランキング**：難易度別の全ユーザーランキング（上位20名を表示）
- スコアの保存・ランキングへの掲載はログインユーザーのみ

## 4. 画面一覧

| 画面名 | URL（案） | 概要 |
|--------|----------|------|
| トップ / ランディング | `/` | アプリ説明・ゲーム開始ボタン・ログインへの導線 |
| ログイン | `/login` | メールアドレス＋パスワード認証フォーム |
| 新規登録 | `/register` | 新規ユーザー登録フォーム |
| 難易度選択 | `/game` | かんたん・ふつう・むずかしいの選択画面 |
| ゲーム画面 | `/game/play` | ヒント表示・タイマー・日本地図・現在スコア |
| 結果画面 | `/game/result` | 正解数・獲得スコア・ベストスコア更新有無・ランキングへの導線 |
| ランキング | `/ranking` | 難易度別の全ユーザーランキング |
| マイページ | `/mypage` | 自分のベストスコア（難易度別）表示 |

## 5. データベース設計（概要）

### `prefectures`（都道府県マスタ）

| カラム名 | 型 | 説明 |
|----------|------|------|
| `id` | int | 主キー |
| `name` | text | 都道府県名（例：東京都） |
| `region` | text | 地方区分（例：関東） |
| `svg_id` | text | 地図SVGの対応ID |

### `hints`（ヒントデータ）

| カラム名 | 型 | 説明 |
|----------|------|------|
| `id` | int | 主キー |
| `prefecture_id` | int | 都道府県ID（外部キー） |
| `difficulty` | text | 難易度（easy / normal / hard） |
| `content` | text | ヒント本文 |

### `scores`（スコア記録）

| カラム名 | 型 | 説明 |
|----------|------|------|
| `id` | uuid | 主キー |
| `user_id` | uuid | ユーザーID（Supabase Auth） |
| `difficulty` | text | 難易度（easy / normal / hard） |
| `score` | int | 獲得スコア |
| `correct_count` | int | 正解数（0〜10） |
| `played_at` | timestamp | プレイ日時 |

> `users` テーブルは Supabase Auth が管理するため別途定義不要

## 6. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブ対応 | PC・スマートフォン両対応（モバイルファーストで設計） |
| パフォーマンス | 地図の初回読み込み 2秒以内を目標 |
| セキュリティ | Supabase RLS（Row Level Security）によるデータアクセス制御 |
| 可用性 | Supabase・Vercel（推奨）のSLAに準拠 |
